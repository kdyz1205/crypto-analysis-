# Trendline Trading Rules — 永久文档，不可删除

## 这是什么

这是 Axel 的交易逻辑定义。任何 AI 或代码在操作 trendline 策略之前必须先读这份文档。
如果你是 AI 正在阅读这段话，在写任何代码之前，确保你的实现 100% 符合下面的规则。

---

## 1. 画线

- 用两个 pivot 点画一条 trendline（两个低点 = 支撑线，两个高点 = 阻力线）
- 向右无限延伸
- 这两个点本身就是线被"踩"了两次
- 线有斜率，有截距，所以你知道**每一根 bar 上这条线的精确坐标**

## 2. 第三次触碰 = 交易机会

- 当价格第三次接近这条线时，我们预期它会反弹（基于历史统计）
- 我们在线的坐标附近**挂一个限价单**

## 3. Buffer — 唯一的距离概念

- 价格不可能 0.000000% 精确地碰到线
- 所以我们在线的上方（做多）或下方（做空）设一个小 buffer
- 例如：线在 100.00，我挂买单在 100.15（buffer = 0.15%）
- **Buffer = 限价单离线的距离**
- **没有单独的 "entry_buffer" 和 "stop_buffer" 两个东西**
- **只有一个 buffer**

## 4. 止损 = 线本身

**这是最关键的一条：**

- 线就是一根线，不是一个厚带
- 价格穿过这条线 = 止损
- 穿过 = 1-2 个 tick 的价格
- **所以：总止损距离 ≈ buffer 本身**
- 如果 buffer = 0.15%，总价格止损 ≈ 0.15%
- **不存在线下面再加一个 "stop_buffer"**
- **线就是止损位，buffer 就是总风险距离**

用数字说明：
```
线坐标 = 100.00
买入限价 = 100.15 (buffer 0.15%)
止损 = 99.99 或 100.00 (线本身，穿过即止损)
总风险 = 100.15 - 100.00 = 0.15%
```

## 5. 账户风险 — 独立于价格距离

- "这一笔我愿意亏多少钱" 是单独定义的
- 例：$1000 账户，这笔愿意亏 4% = $40
- 这和 buffer/止损距离是两码事

## 6. 杠杆 = 账户风险 / 价格止损

**杠杆是算出来的，不是选出来的：**

```
账户风险 = 4% (这笔愿意亏 $40 on $1000)
价格止损 = 0.2% (buffer 决定的)
杠杆 = 4% / 0.2% = 20x
仓位 = $1000 × 20 = $20,000
验证: $20,000 × 0.2% = $40 ✓
```

不同时间框架的例子：
```
3 分钟:
  buffer = 0.05%
  账户风险 = 0.3% per trade ($3 on $1000)
  杠杆 = 0.3% / 0.05% = 6x
  亏 300 笔才扣完 → 高频安全

4 小时:
  buffer = 0.15%
  账户风险 = 4% per trade ($40 on $1000)
  杠杆 = 4% / 0.15% ≈ 26.7x
  亏 25 笔扣完 → 低频大仓
```

## 7. 止盈

- 根据历史回测的同类 trendline 的反弹分布决定
- 或者用 RR 倍数（buffer × N）
- 或者用 BB upper/lower
- 或者用下一个对向 trendline
- 这个可以灵活调

## 8. 历史相似度

- 当前这条线有 slope、anchor distance、age 等特征
- 从历史数据中找结构相似的线
- 看它们第三次触碰后的结果分布
- 用这个分布决定止盈和胜率预期

## 9. 时间调整

- 线有斜率 → 线的坐标每 bar 都在变
- 限价单和止损都应该**跟着线滚动**
- 每根新 bar：重新计算 line_price → 重新设 entry limit 和 stop

## 10. 总结公式

```
line_price_at_bar = slope × bar_index + intercept
entry_price = line_price + buffer  (做多)  或  line_price - buffer  (做空)
stop_price = line_price  (穿过即止损)
total_price_risk = buffer
account_risk_dollars = equity × account_risk_pct_per_trade
position_notional = account_risk_dollars / buffer
leverage = position_notional / equity = account_risk_pct / buffer
target = entry + buffer × RR  或  BB  或  historical analog
```

## Axel 原话 (2026-04-15，一字不差)

限价单离这个线有多远，就是我们的止损。这个线，它只是一根线，这个线不可能有一个 tick 是两个 percent 这么厚。那么你穿过这个线，它就只是一个 tick 价格或者两个 tick 的价格。穿过了马上穿过这个线马上就止损。那不就是说如果我们设置这个 buffer 是在 0.15%，那这个总计的就是 0.15%，因为你只要穿过这个线就止损了。

那为什么会有一个 stop buffer 和一个 entry buffer 两个东西呢？不应该有。

这一笔愿意亏这么多钱，比如说我现在一千美金，那我这笔愿意亏四十美金，40 美金是百分之四，是总账户的钱。然后我这一笔单止损是价格上的止损是 0.2%。那我总计愿意亏 4%，那我就可以开一个 20 倍的杠杆。0.2% 的止损 × 20 倍 = 4%。

三分钟的周期，给它一个每笔亏损 0.3%，那我亏 300 笔我会扣完所有的钱。四小时的就给它放一个两个点或者一个点或者 1.5 个点，或者说三个点四个点，根据每一笔的止损——我说的是每一笔金额的止损，就这一笔愿意亏多少钱。

回测就是让你来计算这些的。
