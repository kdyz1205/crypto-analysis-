什么是 train / test split

历史数据按时间切前 70% 给策略调参 (train), 后 30% 锁起来不让策略看到 (test). gate 只看 test split 的数字 — 这样如果策略在 train 上很漂亮但 test 上拉胯, 我们就 知道是过拟合, 不是真信号.

为什么一个币有多行

每行答一个具体条件下的表现, 不是不同参数. 一个币 BTCUSDT 在 1h+4h 两个 TF, 6 个 距离 MA5 档 (0–0.5%, 0.5–1%, ...), 2 个 split (train/test), 最多 2×6×2 = 24 行. 每行: "BTC 在 1h, 排列形成且距 MA5 在 0–0.5%, test split, 平均收益是 X".

Phase 1 vs Phase 2

Phase 1: 信号 cohort. 测每次"多头排列"成立后, 固定 N 根 bar 内的 平均收益. 不模拟真实交易, 只测信号本身有没有边际.

Phase 2: 真实交易模拟. 信号触发 → 下一根 bar 开盘买入 → SL = 当下 EMA21 × (1 − buffer%), 永远只往上 trail → 价格跌穿 SL 平仓. 输出 胜率 / 总收益 / 回撤 / Sharpe / 持仓时长 等真实策略指标.

手续费 / 资金费率

默认 Bitget taker 0.05% × 进出 + 1bp 滑点 × 进出 = 来回 0.12%. 资金费率 (funding rate) 暂未算入 — 多头持仓 8 小时一次, -0.005% ~ -0.05% 不等. 如果平均持仓 10+ bar on 4h (= 40+ 小时 = 5+ 次资金费率), 影响会显著. 后续会加入.
